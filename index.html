Usability

The user interface of an application is the actual product you are selling.

Usability isn't applied at the end. It is to be considered from the start. It's a large part of the hole product design. A good UI doesn't make its user think. It solves problems, so its user doesn't have to. Simple interfaces are easy to use, easy to understand, intuitive, fast loading, and easy to maintain.

Salutation and country are both a finite set of options. But this doesn’t mean the way of presenting them should be the same. Just because the UI would be more consistent, but the experience could be richer.

Context over consistency.

The options of the salutation are so few. Thus radio buttons are far more appropriate.


Even though the copywriting is clear, the link isn’t placed poorly. While skimming a page, the user tends to identify the links first. In the first case, he/she has to figure out, what “here” acutally is.

Link text should label its destination. Even if taken out of context.

Show state instead of just display it. It helps the user to identify active or inactive records and/or wether action should be taken or not.

  1. This record is discarded. Thus its primary link doesn’t have to be that prominent.
  2. These records are in different phases. They are displayed in associated colors.
  3. This icon indicates a boolean value. Since it’s in its own column and just displayed if true, users are able to pick these records up.
  4. Overdue records are stand out due the loud color.
